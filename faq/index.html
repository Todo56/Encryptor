Frequently Asked Questions:


  • Do you store the data I encrypt?

    No, we will never store your data. In fact, all of the website's code is being executed in your own pc so you can inspect the website to make sure your data remains secure. This also means that if you loose the password or the encryption algorithm of your data it's gone forever!

  • What files can I encrypt?

    It is recommended that you only encrypt plain text data, this means, no word, excel or image documents. Encrypting them will corrupt them and make them unusable!

    Tutorial

  • What are unsecure algorithms?

    An encryption algorithm is unsecure when it repeats too many characters or generates encrypted strings that are too short. These are not currently cracked yet but they aren't recommended either.

  • What is the difference between hashing and encrypting?

    Hashing is practically converting a string into another string that cannot be reverted, this is mainly used for storing passwords (this is why you cannot request a website to send you your password) and other functions that require it. Encryption is converting a string into another string using a password.

    These explanations are very vague, research in your own if you are interested.
